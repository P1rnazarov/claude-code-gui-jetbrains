# 멀티 계정 관리 (Multi-Account Management)

> 언어: [English](./en.md) · **한국어**
>
> 관련: [PR #134](https://github.com/yhk1038/claude-code-gui-jetbrains/pull/134)

## 새로운 기능

이제 Claude Code with GUI에서 **여러 개의 Claude 계정을 저장해두고, 클릭 한 번으로 전환**할 수 있습니다. 회사 계정과 개인 계정을 오가거나, 플랜이 다른 계정을 함께 쓰는 경우에 매번 로그아웃·재로그인할 필요가 없습니다.

- **Settings → Account**: 저장된 계정을 한눈에 보고, 전환·추가·삭제합니다.
- **상단바 계정 아바타**: 채팅 화면 상단의 아바타 버튼을 눌러 빠르게 계정을 전환합니다.
- **Settings → Usage**: 저장된 **모든 계정**의 사용량을 한 화면에서 비교합니다.

## 계정 추가와 자동 저장

- **Add account**: Settings → Account 우상단(또는 상단바 드롭다운)의 **Add account** 를 누르면 앱 안에서 로그인 절차가 진행되고, 로그인에 성공하면 그 계정이 **자동으로 목록에 저장**됩니다.
- **현재 계정 자동 등록**: 터미널에서 `claude login` 으로 이미 로그인해 둔 계정이 목록에 없다면, **Settings → Account 화면을 열 때 자동으로 등록**됩니다. 따로 "저장" 버튼을 누르지 않아도 됩니다.

## 계정 전환

- **상단바 아바타 → 드롭다운**: 다른 계정 줄을 클릭하면 즉시 전환되고, 전환이 끝나면 화면 상단에 알림(토스트)이 뜹니다.
- **Settings → Account → Switch**: 목록의 각 계정 줄에 있는 **Switch** 버튼으로도 전환할 수 있습니다. 현재 사용 중인 계정은 **In use** 로 표시됩니다.
- 전환은 시스템에 실제로 로그인된 자격증명을 **물리적으로 교체**합니다 (macOS는 Keychain, Linux/Windows는 `.credentials.json`). 교체에 실패하면 이전 상태로 자동 롤백되어 로그인이 깨지지 않습니다.

각 계정 줄에는 아바타, 이름, **플랜**과 **인증 방식** 배지(마우스를 올리면 무엇인지 알려주는 툴팁), 이메일, 그리고 **마지막 접속 시각**이 함께 표시됩니다.

## 계정별 사용량 (Settings → Usage)

사용량 페이지는 이제 활성 계정뿐 아니라 **저장된 모든 계정의 사용량**을 동시에 보여줍니다. 각 계정마다 5시간 세션 한도와 주간(7일) 한도, 그리고 초기화 시각을 확인할 수 있고, 계정 우측에 플랜 배지가 표시됩니다.

- **활성 계정**: 현재 로그인된 계정의 사용량은 실시간으로 조회됩니다.
- **비활성 계정**: 마지막으로 그 계정이 **활성 상태였을 때 조회해 둔 사용량**을 보여줍니다. 따라서 값이 다소 과거의 것일 수 있으며, 해당 계정으로 전환하면 최신 값으로 갱신됩니다. 아직 한 번도 활성화된 적이 없는 계정은 *"Switch to this account to load its usage data."* 안내가 표시됩니다.

## 보안과 프라이버시

- **OAuth 토큰·자격증명은 어디에도 로그로 남지 않습니다.**
- 사용량 조회는 **현재 활성 계정에 대해서만** 이루어지며, 그 외 계정은 위에서 설명한 캐시된 값을 사용합니다. 즉, 비활성 계정 때문에 백그라운드에서 외부로 추가 요청이 나가지 않습니다.
- 사용량은 계정당 5분간 캐시되어, 메시지를 보낼 때마다 불필요하게 다시 조회하지 않습니다. **Usage 페이지의 새로고침 버튼**으로 언제든 최신 값을 강제로 가져올 수 있습니다.

## 알아두기

- 계정 목록은 **등록한 순서**대로 정렬되며, 전환을 하더라도 순서가 바뀌지 않습니다.
- 저장된 계정은 기기 내부(`~/.claude-code-gui/accounts/`)에만 보관됩니다.
